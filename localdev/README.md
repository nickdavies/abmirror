# Local Development Environment

Run Actual Budget in Docker and bootstrap from your exported budget zips for testing ab-mirror pipelines.

## Prerequisites

- Docker (or docker compose)
- Node.js 22+
- 3 Actual Budget export files (.zip from the UI)

## Server

- **URL:** `http://localhost:5007`
- **Password:** `test`

## Quick Start

```bash
# 1. Bootstrap from 3 zips (starts Docker, imports, writes config)
# Zip paths are relative to your current directory, or use absolute paths
cd localdev && ./bootstrap.sh nick.zip britta.zip joint.zip

# Encrypted exports are uploaded unencrypted (export contains decrypted data).
# No keys needed for bootstrap.

# 2. Run a pipeline
./run-pipeline.sh --pipeline pipelines/split-only.yaml
./run-pipeline.sh --pipeline my-experiment.yaml --dry-run
```

## Commands

| Command | Description |
|---------|-------------|
| `./bootstrap.sh [--port 5007] zip1 zip2 zip3` | Start Docker, import zips, write `config.yaml` |
| `./teardown.sh [--clean]` | Stop Docker containers; use `--clean` to also remove `./data` and `config.yaml` |
| `./run-pipeline.sh --pipeline FILE \| -` | Merge pipeline with config, run ab-mirror |
| `./run-pipeline.sh --config PATH` | Use a different base config (default: `config.yaml`) |
| `./run-pipeline.sh --dry-run` | Simulate without writing |
| `./run-pipeline.sh --step N` | Run only pipeline step N (1-based) |

## File Layout

- `docker-compose.yml` - Actual server on port 5007, persistent `./data`
- `bootstrap.sh` - Entry script
- `teardown.sh` - Stop containers (optionally `--clean` to remove data and config)
- `bootstrap.ts` - Imports zips via Actual API
- `run-pipeline.sh` - Merges pipeline YAML, runs ab-mirror
- `config.yaml` - Generated by bootstrap (gitignored)
- `pipelines/` - Example pipeline snippets
